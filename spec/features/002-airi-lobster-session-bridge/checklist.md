# 验收清单：Airi × LobsterAI 会话化 Agent Bridge 重构

**Feature**: 002-airi-lobster-session-bridge  
**创建时间**: 2026-03-27

---

## 当前说明（2026-04-02）

- 下列清单保持“验收视角”，只有完成联调或具备明确验证证据后才建议勾选
- 当前实现完成度已明显高于勾选状态：协议、技能、权限与回退主链已基本落地
- 当前主要缺口不是继续堆功能，而是补齐 Phase 8 的系统化验证、会话恢复收口与文件回收
- 2026-04-02 已新增一轮服务端级 Bridge 真联调：session bind、多轮续聊、提示词透传、`fileId` 文件读取、技能接口均已通过；Airi 前端 UI 仍需先切到 `lobster-agent` provider 后再做界面验收

---

## 一、协议与会话

- [ ] Airi 侧每个聊天会话都能绑定唯一的 Lobster runtime session
- [ ] 同一会话的后续追问走 `continueSession`，而不是重新创建 session
- [ ] Bridge 事件均包含 `sessionId`、`turnId`、`seq`、`type`、`createdAt`
- [ ] 页面刷新后仍可恢复会话绑定关系
- [ ] 旧会话事件不会污染新会话 UI

## 二、流式回复

- [ ] 发送消息后，Airi 能在回复过程中逐步显示文本
- [ ] 不再出现“整段完成后一次性显示”或“文本突然消失”
- [ ] 最终展示内容与 Lobster 最终消息对账一致
- [ ] tool call/result 不会混入最终文本正文
- [ ] 出现错误时能显示结构化错误而不是静默失败

## 三、动作与状态

- [ ] 发送后、首 token 前，Airi 进入等待态动作
- [ ] 收到首个流式增量后，Airi 进入说话态并驱动口型
- [ ] 工具执行时，Airi 能感知工具中状态
- [ ] 回复完成后，Airi 能恢复完成态或 idle
- [ ] 动作主驱动来自结构化状态事件，而不是仅依赖 ACT 文本

## 四、思考与过程

- [ ] 用户可以看到思考态或过程态，而不是只看到最终回答
- [ ] 过程态与最终回答分层显示，不互相污染
- [ ] 过程事件不会导致最终聊天记录错乱

## 五、技能

- [ ] Airi 能拉取技能列表
- [ ] Airi 能显示技能启用状态
- [ ] Airi 能启停技能
- [ ] Airi 能勾选 turn 级技能并随消息发送
- [ ] Airi 能查看技能详情和配置项
- [ ] Airi 能修改技能配置并在 LobsterAI 中生效

## 六、文件与图片

- [ ] 图片附件可以被识别并参与回复
- [ ] 普通文件可以上传并在同一会话中引用
- [ ] Airi 侧只持有 `fileId` 或元数据，不直接持有物理路径
- [ ] LobsterAI 可正确把 `fileId` 解析到会话内文件
- [ ] 文件过期或会话结束后可被回收

## 七、权限请求

- [ ] LobsterAI 触发权限请求时，Airi 能收到结构化请求
- [ ] Airi 能正确返回 allow/deny 结果
- [ ] 错误会话或错误窗口不能响应不属于自己的权限请求
- [ ] 超时未响应时，权限请求默认安全拒绝

## 八、安全与隔离

- [ ] 文件上传接口不返回绝对路径
- [ ] Session 事件不会广播给所有无关窗口
- [ ] 权限响应携带会话归属与一次性能力标识
- [ ] 本机 API 不暴露多余通用调用能力
- [ ] Bridge 故障时回退路径明确且不会泄露高阶能力

## 九、回退与兼容

- [ ] `/v1/chat/completions` 文本兼容接口仍可工作
- [ ] 关闭 Bridge 后，系统可回退为纯文本模式
- [ ] 回退模式下，技能深接入、文件会话化、权限闭环会被显式关闭
- [ ] 回退模式不会破坏已有普通 provider 的聊天能力
