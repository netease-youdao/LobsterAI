# LobsterAI 核心功能里程碑（当前阶段）

## 1. 范围声明

- 本阶段聚焦核心能力：OpenAI 兼容链路稳定、AIRI 动画联动、核心执行质量。
- 手机 IM 触发桌面 Agent 属于冻结范围，不纳入本阶段里程碑验收。

## 2. 里程碑总览

| 里程碑 | 时间窗口 | 目标 | 退出条件 |
|---|---|---|---|
| M1 稳定化 | Week 1-2 | 主链路稳定 + 五态动画闭环 | P0 全通过，主链路 3 天零阻断 |
| M2 治理化 | Week 3 | 文档与准入流程固化 | roadmap/milestones/dod 落地并执行 |
| M3 可观测 | Week 4 | 定位效率提升 | 核心故障 10 分钟内可定位 |

## 3. M1 交付清单

1. OpenAI 兼容链路回归集：
   - `/v1/models`、`/v1/chat/completions`
   - 鉴权头兼容
   - 流式输出完整性
2. AIRI 动画协议 v1 接入：
   - 五态事件联动（think/tool_use/ask_user/success/error）
   - 回退策略生效
3. 噪音收敛：
   - 未使用 provider 不再刷阻断级错误
4. 稳定性指标：
   - 事件触发成功率 >= 99%
   - 会话无异常堆积

## 4. M2 交付清单

1. 规划资产齐备：
   - `roadmap-v1.md`
   - `milestones.md`
   - `dod.md`
2. 发布准入流程明确：
   - 合并前检查项固定
   - 风险回滚流程固定
3. 基线配置模板：
   - 本地开发模板
   - 联调自检脚本

## 5. M3 交付清单

1. 观测面板：
   - 主链路健康状态
   - 动画事件成功率
   - 会话与任务执行统计
2. 日志分级：
   - 阻断错误 / 可忽略告警 分层
3. 故障演练：
   - 典型失败场景恢复时长 < 10 分钟

## 6. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 事件协议不稳定 | 动画乱跳、体验退化 | 协议版本化 + 去抖策略 + 回退机制 |
| 旧配置残留 | 控制台噪音与误报 | 启动自检 + 配置归一化 |
| 回归覆盖不足 | 上线后回归缺陷 | 增加最小回归集并设为准入门槛 |

## 7. 每周节奏

1. 周一：目标确认与风险评审
2. 周三：中期对齐与阻塞清理
3. 周五：验收、复盘、更新文档
