## 任务列表分类（文件夹功能）实现说明

### 功能概述

用户可以将侧边栏中的任务（会话）归类到自定义文件夹中，方便在任务数量较多时进行分组管理。文件夹名称持久化存储于本地 SQLite 数据库，重启应用后不丢失。

### 数据流

```
用户右键任务 → 菜单"移动到文件夹" → 鼠标悬停展开二级子菜单
  → 选择已有文件夹 / 输入新文件夹名 / 移出文件夹
  → coworkService.setSessionFolder(sessionId, folder)
  → IPC: cowork:session:setFolder
  → main/coworkStore.setSessionFolder(id, folder)
       └─ UPDATE cowork_sessions SET folder = ? WHERE id = ?
  → dispatch updateSessionFolder({ sessionId, folder })
       └─ Redux state.sessions[i].folder = folder
  → CoworkSessionList 重新分组渲染（folderGroups useMemo 更新）
```

---

### 数据库迁移说明

迁移逻辑位于 `SqliteStore.migrate()` 方法，与其他字段迁移（`execution_mode`、`pinned`、`active_skill_ids` 等）保持一致的模式：

```typescript
// src/main/sqliteStore.ts
const colsResult = this.db.exec("PRAGMA table_info(cowork_sessions);");
const columns = colsResult[0]?.values.map((r) => r[1] as string) ?? [];

if (!columns.includes('folder')) {
  this.db.run("ALTER TABLE cowork_sessions ADD COLUMN folder TEXT NOT NULL DEFAULT '';");
}
```

`DEFAULT ''` 保证存量数据不受影响，升级后所有旧任务的 `folder` 字段自动为空字符串（即"无分类"）。

### 子菜单交互细节

- 鼠标移入"移动到文件夹"菜单项 → 右侧弹出子菜单（150ms 防抖延迟关闭，防止鼠标移动过程中子菜单闪烁消失）
- 子菜单优先向右展开，空间不足时自动改向左展开
- 当前任务已在某文件夹中，对应条目显示 ✓ 勾选标记并禁用点击
- "新建文件夹"弹出居中对话框，输入框自动获焦，`Enter` 确认、`Escape` 取消，确定按钮在输入为空时禁用
