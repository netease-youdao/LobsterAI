# LobsterAI Roadmap v1（基线与完成度）

## 1. 基线说明

- 本文基于最初公开规划（README 核心特性）与当前仓库实现状态对齐整理。
- 目标：明确哪些能力已完成、哪些待补齐、哪些属于后续迭代。
- 评估时间：2026-03-06。
- 当前阶段范围裁剪：暂停“手机 IM 触发桌面 Agent”的继续开发与扩展，仅保留既有能力，不作为本阶段交付目标。

## 2. 规划基线（来自 README）

### 2.1 核心特性基线

1. 全场景办公助理  
2. 本地 + 沙箱执行  
3. 内置技能  
4. 定时任务  
5. 持久记忆  
6. IM 远程操控  
7. 权限门控  
8. 跨平台  
9. 数据本地化  

### 2.2 关键场景基线

1. 对话式/GUI 定时任务创建  
2. 本地数据持久化与历史可追溯  
3. 可扩展技能生态  
4. AIRI 动画表情与任务反馈联动

## 4. 已完成事项（可复用资产）

### 4.1 主流程与系统能力

- Electron 主进程 + 渲染进程隔离架构。
- CoworkRunner 本地/沙箱执行与权限审批。
- Scheduled Task Store + Scheduler 全链路。
- IM 网关统一管理（DingTalk/Feishu/Telegram/Discord/NIM）。
- SQLite 结构化持久化（会话、消息、记忆、定时任务）。

### 4.2 技能生态

- 默认技能集覆盖文档处理、搜索、自动化、视频、天气、邮件、计划、技能创建等。
- 技能以独立目录组织，具备脚本/文档/配置结构，支持增量扩展。

### 4.3 本轮联调新增（AIRI × Lobster）

- OpenAI 兼容路径增强（models/chat 路由兼容）。
- 鉴权头兼容与本地 key 规范化。
- 流式输出补齐 messageUpdate，修复短回复问题。
- Provider 校验噪音收敛（减少未使用 provider 的连接报错）。

## 5. 未完成/待补齐项

### 5.1 规划治理层

- 缺少独立版本化规划文档（PRD/Roadmap/Milestone/DoD）。
- 缺少统一“已完成定义”与发布准入清单。
- 缺少跨模块联调验收脚本（桌面端 + AIRI 端一键验证）。

### 5.2 工程质量层

- 需要补一套稳定的端到端回归用例（OpenAI compatible + streaming + scheduled task）。
- 需要持续清理非阻断日志噪音与观测面板分级。
- 需要建立“配置残留探测”机制，避免历史 provider 配置影响体验。
- 需要建立 AIRI 动画状态机与 Skill 事件映射规范（任务开始/思考/调用工具/成功/失败/等待输入）。

### 5.3 体验联动层（AIRI × Lobster）

- 缺少统一的“任务语义 -> 动画表情”映射表。
- 缺少跨端事件总线（Lobster 执行事件推送到 AIRI 动画层）。
- 缺少分任务类型反馈模版（搜索、文档生成、代码执行、失败重试）。
- 缺少动画回退策略（模型不支持特定动作时如何降级）。

### 5.4 范围冻结项（本阶段不做）

- 不新增 IM 平台接入能力。
- 不推进手机 IM 触发桌面 Agent 的新功能迭代。
- 不将 IM 链路作为本阶段里程碑验收阻断项。

## 6. 变更说明（相对初始规划）

- 初始规划聚焦桌面 Agent 主体；现已实质扩展到 AIRI 生态互联与 OpenAI 兼容桥接。
- 初始规划强调“全场景 + 安全门控”；当前实现已覆盖，但“可观测性/回归自动化/规划治理”仍是短板。

## 7. 下一阶段建议（v1.1）

### P0（必须）

1. 建立联调回归套件：`models / chat / stream / auth / scheduler`。
2. 建立规划资产：`docs/roadmap-v1.1.md + docs/milestones.md + docs/dod.md`。
3. 固化配置模板：本地开发默认 provider 模板与自检脚本。
4. 定义 AIRI 动画联动协议 v1：`state + emotion + duration + fallback`。
5. 打通最小闭环：`Think / ToolUse / Success / Error / AskUser` 五态动画反馈。
6. 明确 IM 冻结策略：仅维护现有稳定性，不扩展新需求。

### P1（重要）

1. 完成日志分级：阻断错误与可忽略告警分层展示。
2. 增加健康探针页：展示各子系统当前可用性。
3. 优化记忆与定时任务的可观测 UI（命中率、失败原因、最近执行链路）。
4. 增加按任务类型的动画模版（文档类、搜索类、代码类、语音类）。

### P2（增强）

1. 扩展技能质量门禁（每个技能最小测试与 smoke 检查）。
2. 增强跨平台安装后自检流程（mac/win/linux 首次启动诊断）。

## 8. 里程碑建议

| 里程碑 | 目标 | 验收标准 |
|---|---|---|
| M1（稳定化） | 联调链路稳定 + 动画联动最小闭环 | 连续 3 天主链路零阻断，五态动画事件触发成功率 >= 99% |
| M2（治理化） | 文档化与发布准入规范 | roadmap/milestone/dod 三文档齐备并执行 |
| M3（可观测） | 故障定位效率提升 | 主要故障可在 10 分钟内完成定位 |

## 9. 使用方式

- 作为后续规划评审与迭代排期的基线文件。
- 每次迭代结束后更新“完成度总览”和“里程碑状态”。
